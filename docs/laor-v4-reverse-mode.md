# 라오어 무한매수법 V4.0 교본 - 소진후 리버스모드

> 내부 프로젝트 정리 문서입니다. 사용자가 제공한 소진후 리버스모드 자료를 교본 형태로 재구성했습니다. 외부 배포용 문서가 아닙니다.

## 범위

이 문서는 라오어 무한매수법 V4.0의 **소진후 리버스모드**를 다룹니다.

일반모드는 소진 전까지의 매수, 쿼터매도, 최종매도 과정을 뜻합니다. 리버스모드는 진행 회차 `T` 기준으로 1회분 미만이 남은 시점부터 시작하고, 종료 조건을 만족하면 다시 일반모드로 돌아갑니다.

일반모드는 [laor-v4-normal-mode.md](./laor-v4-normal-mode.md)를 참고합니다.

## 리버스모드 발동 조건

리버스모드는 소진이 발생하면 시작합니다.

분할 수별 기준:

| 분할 수 | 리버스모드 진입 기준 |
| --- | --- |
| `20분할` | `T > 19` |
| `40분할` | `T > 39` |

일반화하면 다음과 같습니다.

```text
T > N - 1
```

남은 현금이 있더라도 위 조건을 만족하면 리버스모드로 넘어갑니다. 남은 현금은 리버스모드 매수 과정에서 그대로 사용합니다.

## 리버스모드 핵심 변수

| 변수 | 의미 |
| --- | --- |
| `N` | 전체 분할 수 |
| `T` | 진행 회차 |
| `cash` | 남은 현금 |
| `avg` | 평균 매수가 |
| `qty` | 현재 보유 수량 |
| `targetProfitPercent` | `TQQQ = 15%`, `SOXL = 20%` |
| `reverseStarPoint` | 직전 5거래일 종가 평균 |

## 리버스 별지점

리버스모드의 별지점은 일반모드의 별% 공식으로 계산하지 않습니다.

```text
reverseStarPoint = 직전 5거래일 종가 평균
```

예를 들어 한국 기준 월요일 본장 시작 전이면, 직전 주 월요일부터 금요일까지의 미국장 종가 평균을 사용합니다.

첫날 매도는 무조건 매도를 의도하므로 별지점 없이 `MOC` 매도를 사용합니다. 둘째 날부터는 리버스 별지점을 기준으로 매도와 매수를 나눕니다.

리버스모드에서 `T`는 리버스 별지점 계산에 사용하지 않습니다. `T`는 리버스 매수/매도 후 일반모드로 복귀할 때 이어서 사용할 진행 회차입니다.

## 리버스 매도 규칙

### 첫날 매도

리버스모드 첫날에는 매수하지 않고 매도만 시도합니다.

규칙:

- `20분할`: 전체 보유 수량을 `10등분`
- `40분할`: 전체 보유 수량을 `20등분`
- 자연수로 떨어지지 않으면 내림
- 주문 타입은 `MOC`

일반화하면 매도 수량은 다음과 같습니다.

```text
reverseSellQuantity = floor(qty / (N / 2))
```

예:

| 분할 수 | 보유 수량 | 매도 분모 | 첫날 매도 수량 |
| --- | --- | --- | --- |
| `20분할` | `200` | `10` | `20` |
| `20분할` | `198` | `10` | `19` |
| `40분할` | `200` | `20` | `10` |
| `40분할` | `198` | `20` | `9` |

### 둘째 날 이후 매도

둘째 날부터는 매일 리버스 별지점 위에서 매도를 시도합니다.

규칙:

- 주문 타입은 `LOC`
- 가격은 `reverseStarPoint`
- 매도 수량은 직전 보유 수량 기준으로 다시 계산

```text
reverseSellQuantity = floor(currentQty / (N / 2))
```

예를 들어 `40분할`, 소진 시 보유 수량 `200`인 경우:

| 날짜 | 시작 보유 수량 | 매도 수량 계산 | 시도 매도 수량 | 매도 후 보유 수량 예 |
| --- | --- | --- | --- | --- |
| 첫날 | `200` | `floor(200 / 20)` | `10` | `190` |
| 둘째 날 | `190` | `floor(190 / 20)` | `9` | `181` |
| 셋째 날 | `181` | `floor(181 / 20)` | `9` | `172` |
| 넷째 날 | `172` | `floor(172 / 20)` | `8` | `164` |

이 방식으로 보유 수량 기준 매도량을 계속 갱신합니다.

## 리버스 매수 규칙

리버스모드의 매수는 쿼터매수입니다.

### 첫날

첫날은 매수하지 않습니다. `MOC` 매도만 시도합니다.

### 둘째 날 이후

둘째 날부터는 총 잔금의 `1/4`만큼 매수 시도합니다.

```text
reverseBuyBudget = cash / 4
```

가격:

```text
reverseBuyPrice = reverseStarPoint - 0.01
```

주문 타입:

```text
LOC
```

매수 체결 여부와 매도 체결 여부와 관계없이, 종료 조건을 만족하기 전까지 매일 현재 잔금을 기준으로 `cash / 4`를 다시 계산합니다.

## 리버스모드 T값

리버스모드에서도 T값을 계속 갱신합니다. 이 값은 리버스모드 종료 후 일반모드로 돌아갈 때 그대로 이어서 사용합니다.

분할 수별 공식:

| 분할 수 | 리버스 매도 체결 | 리버스 매수 체결 |
| --- | --- | --- |
| `20분할` | `T = 직전 T * 0.9` | `T = 직전 T + (20 - 직전 T) * 0.25` |
| `40분할` | `T = 직전 T * 0.95` | `T = 직전 T + (40 - 직전 T) * 0.25` |

일반식:

```text
리버스 매도 후 T = T * (1 - 2 / N)
리버스 매수 후 T = T + (N - T) * 0.25
```

예: `40분할`, 리버스 진입 시 `T = 39.5`

```text
첫날 매도 후 T = 39.5 * 0.95 = 37.525
둘째 날 매수 후 T = 37.525 + (40 - 37.525) * 0.25 = 38.14375
```

일반모드와 동일하게 T값의 소수점 자리수는 제한하지 않습니다.

## 리버스모드 종료 조건

리버스모드는 종가가 평균단가 대비 손실폭을 충분히 회복하면 종료합니다.

종목별 종료 기준:

| 종목 | 종료 조건 |
| --- | --- |
| `TQQQ` | `close > avg * 0.85` |
| `SOXL` | `close > avg * 0.80` |

일반식:

```text
close > avg * (1 - targetProfitPercent / 100)
```

예를 들어 `SOXL` 평균단가가 `40`이면:

```text
종료 기준 = 40 * 0.80 = 32
```

종가가 `32`를 넘는 것을 확인하면 리버스모드를 종료하고 일반모드로 돌아갑니다. 상황에 따라 리버스모드가 하루 만에 종료될 수도 있습니다.

리버스모드 동안의 매수와 매도는 모두 손절 성격을 가질 수 있습니다. 규칙을 임의로 중단하거나 외부 자금을 투입하면 교본 기준의 무한매수법이 아니라 개인 변형으로 봅니다.

## 일반모드 복귀

리버스모드 종료 후에는 일반모드로 돌아갑니다.

복귀 규칙:

- 리버스모드에서 계산하던 `T`를 그대로 이어서 사용합니다.
- 일일 1회 매수금은 일반모드와 동일하게 계산합니다.

```text
일일 1회 매수금 = cash / (N - T)
```

복귀 후 다시 `T > N - 1`이 되면 리버스모드가 다시 시작됩니다.

## 자금 추가 투입 원칙

리버스모드 중간에 외부 자금을 추가 투입하지 않습니다.

전략 시작 직전에 설정한 원금 안에서 일반모드와 리버스모드를 모두 해결하는 것이 기본 원칙입니다. 손실이 싫더라도 리버스모드 규칙을 유지해야 교본 기준의 무한매수법으로 볼 수 있습니다.

## 이후 정리 대상

다음 항목은 별도 교본 또는 운영 절차로 분리합니다.

- 리버스모드 종료 판단 시점: 종가 확인 후 당일 즉시 일반모드로 볼지, 다음 거래일부터 일반모드로 볼지
- 리버스 첫날 `MOC` 매도 후 당일 종료 조건도 평가할지 여부
- 리버스 LOC 매수/매도 동시 체결 시 T 업데이트 순서
- 5거래일 종가가 부족한 초반 데이터 구간 처리
- 실제 증권사별 `MOC`, `LOC` 주문 가능 시간과 예약 주문 정책
